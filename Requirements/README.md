# Projekt-Requirements (High-Level)

Zweck: Wasserfester Bauplan für die App – Datenmodell, Prinzipien, Integrationspfade. Diese Requirements sind maßgeblich für alle zukünftigen Entscheidungen (Code, DB, API, UI).

Inhalte in diesem Ordner:
- `db-schema.mmd`: Mermaid-ERD der Zieldatenbank
- `spec.md`: Detaillierte technische Spezifikation (Tabellen, Felder, Constraints, APIs, Migrationshinweise)
- `rules.md`: Verbindliche Projektregeln (Designprinzipien, Sicherheitsvorgaben, Homebrew/Canon-Policy, Testing/Review)

Hinweis: Bevor Implementierungen vorgenommen werden, sind Änderungen hier (Pull-Request / Review) zu diskutieren und zu beschließen.
